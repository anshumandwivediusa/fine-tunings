# Module 2: Foundation of Neural Networks

## Overview

Before learning how to fine-tune a Large Language Model (LLM), it is essential to understand **how neural networks learn**. Fine-tuning is nothing more than continuing the training of a pre-trained neural network on a new dataset. To appreciate what happens during fine-tuning, you must first understand the learning process behind every neural network.

Neural networks learn by making predictions, measuring how far those predictions are from the correct answers, and gradually adjusting their internal parameters to reduce errors. This learning cycle is repeated thousands or even millions of times until the model can generalize well to unseen data.

In this module, you'll explore the fundamental concepts that drive the learning process of modern deep learning models, including Transformers and Large Language Models (LLMs).

## 2.1 How Neural Networks Learn

### **Biological inspiration**
Neural networks are loosely inspired by the human brain. Just as biological neurons transmit signals through synapses, artificial neurons pass information through weighted connections. The analogy helps us design systems that can “learn” patterns from data.


### **Artificial neurons**
- Each artificial neuron takes inputs, multiplies them by **weights**, adds a **bias**, and applies an **activation function**.  
- This determines whether the neuron “fires” and passes information forward.  
- Common activations: sigmoid, ReLU, tanh.  


### **Layers in a neural network**
- **Input layer**: Receives raw data (pixels, words, features).  
- **Hidden layers**: Transform inputs through nonlinear functions, extracting features.  
- **Output layer**: Produces predictions (classification labels, regression values).  


### **Parameters (Weights & Biases)**
- **Weights**: Control the strength of connections between neurons.  
- **Biases**: Allow shifting of activation functions for flexibility.  
- Together, they are the **trainable parameters** updated during learning.  


### **Learning from examples**
- Neural networks learn by comparing predictions to actual labels.  
- Errors are measured using a **loss function**.  
- Through **backpropagation** and **gradient descent**, weights and biases are adjusted to minimize error.  
- Over many **epochs**, the network generalizes patterns from training data to unseen examples.  


📊 **Conceptual Flow**  
Input → Weighted sum + Bias → Activation → Forward Propagation → Loss → Backpropagation → Weight updates → Improved predictions  


## 2.2 Forward Propagation

### **Input layer**
- Accepts raw data (e.g., pixel values, word embeddings, numerical features).  
- Each input is represented as a vector or matrix that feeds into the network.  


### **Hidden layers**
- Intermediate layers that transform inputs into higher-level features.  
- Each neuron computes a weighted sum of inputs, adds a bias, and applies an activation function.  
- Multiple hidden layers allow deep networks to capture complex patterns.  


### **Output layer**
- Produces the final prediction.  
- For classification → softmax activation gives class probabilities.  
- For regression → linear activation outputs continuous values.  


### **Matrix multiplication**
- Forward propagation is essentially a series of matrix multiplications:  
  \[
  z = W \cdot x + b
  \]  
- Where \(W\) = weights, \(x\) = input vector, \(b\) = bias.  
- This operation is repeated layer by layer.  


### **Activation functions**
- Introduce non-linearity, enabling networks to learn complex mappings.  
- Common choices:  
  - **ReLU** → efficient, avoids vanishing gradients.  
  - **Sigmoid** → maps values to [0,1], useful for probabilities.  
  - **Tanh** → maps values to [-1,1], centered around zero.  


### **Prediction generation**
- After passing through all layers, the network outputs predictions.  
- Example:  
  - Image classification → “cat” vs “dog” probability.  
  - Sentiment analysis → “positive” vs “negative” score.  
  - Regression → predicted house price.  


📊 **Conceptual Flow**  
Input → Matrix multiplication → Bias addition → Activation → Hidden layers → Output layer → Prediction  


## 2.3 Loss Functions

### **What is loss?**
Loss is a numerical measure of how far a model’s predictions deviate from the actual target values. It quantifies the “error” in learning.

### **Why loss is needed**
- Without loss, the model has no signal to improve.  
- Loss guides **backpropagation** by telling the optimizer which direction to adjust weights.  
- It ensures the model learns patterns instead of memorizing noise.  

### **Measuring prediction error**
- Predictions are compared against ground truth labels.  
- The difference (error) is calculated using a **loss function**.  
- Smaller loss → better performance. Larger loss → poor predictions.  

### **Common loss functions**
Different tasks require different loss functions:

- **Mean Squared Error (MSE)**  
  - Used in regression tasks.  
  - Formula:  
    \[
    L = \frac{1}{n} \sum (y_{true} - y_{pred})^2
    \]  
  - Penalizes large errors more heavily.  

- **Binary Cross Entropy**  
  - Used for binary classification (yes/no, true/false).  
  - Formula:  
    \[
    L = -\frac{1}{n} \sum \big[y \cdot \log(p) + (1-y) \cdot \log(1-p)\big]
    \]  
  - Measures how well predicted probabilities match true labels.  

- **Categorical Cross Entropy**  
  - Used for multi-class classification.  
  - Formula:  
    \[
    L = -\sum y_{true} \cdot \log(y_{pred})
    \]  
  - Encourages the model to assign high probability to the correct class.  


📊 **Quick Summary Table**

| **Loss Function** | **Task Type** | **Key Feature** |
|-------------------|---------------|-----------------|
| **MSE** | Regression | Penalizes large errors |
| **Binary Cross Entropy** | Binary classification | Probability-based error |
| **Categorical Cross Entropy** | Multi-class classification | Encourages correct class probability |


## 2.4 Backpropagation

### **Error propagation**
- After **forward propagation**, the model produces predictions.  
- The difference between predictions and actual labels is measured by the **loss function**.  
- Backpropagation sends this error signal backward through the network to adjust parameters.  


### **Chain Rule**
- Backpropagation relies on the **chain rule of calculus** to compute derivatives layer by layer.  
- Each gradient is calculated by multiplying partial derivatives of the current layer with those of the next.  
- This allows efficient computation of how each weight contributed to the final error.  


### **Computing gradients**
- Gradients represent the slope of the loss function with respect to each parameter.  
- For each weight \(w\):  
  \[
  \frac{\partial L}{\partial w}
  \]  
  tells us how much changing \(w\) will affect the loss.  
- These gradients are used to update weights in the direction that reduces error.  


### **Weight updates**
- Once gradients are computed, the optimizer (e.g., SGD, Adam) updates weights:  
  \[
  w_{new} = w_{old} - \eta \cdot \frac{\partial L}{\partial w}
  \]  
- Here, \(\eta\) = learning rate.  
- This iterative process gradually improves predictions.  


### **Why gradients matter**
- Gradients are the **learning signal** for the network.  
- Without them, the model cannot adjust weights to minimize loss.  
- Proper gradient flow ensures the network learns efficiently; poor flow leads to issues like **vanishing gradients** or **exploding gradients**.  


📊 **Conceptual Flow**  
Forward pass → Loss calculation → Backward pass (gradients via chain rule) → Weight updates → Improved predictions

# The Neural Network Learning Pipeline

Every neural network follows the same learning cycle during training:

```text
Training Data
      │
      ▼
Forward Propagation
      │
      ▼
Prediction
      │
      ▼
Calculate Loss
      │
      ▼
Backpropagation
      │
      ▼
Compute Gradients
      │
      ▼
Optimizer Updates Weights
      │
      ▼
Repeat for Every Batch
      │
      ▼
Next Epoch
```

This process continues until the model minimizes its prediction error or reaches the desired performance.

# Understanding the Core Components

## Forward Propagation

Forward propagation is the process of passing the input data through the neural network to generate predictions. At each layer, the network applies mathematical operations using learned weights, biases, and activation functions to transform the input into increasingly meaningful representations.

**Goal:** Generate a prediction based on the current model parameters.


## Loss Function

A loss function measures how far the model's prediction is from the actual target value. A lower loss indicates better predictions, while a higher loss signals that the model needs to improve.

**Goal:** Quantify prediction error.


## Backpropagation

Backpropagation computes how much each weight contributed to the prediction error by calculating gradients. These gradients indicate the direction and magnitude of change needed to reduce the loss.

**Goal:** Determine how to adjust the model's parameters to improve future predictions.


## Gradient Descent

Gradient descent is the optimization algorithm that updates the model's weights using the gradients computed during backpropagation. By repeatedly moving the parameters in the direction that reduces the loss, the model gradually learns the underlying patterns in the data.

**Goal:** Minimize the loss function over time.


## Optimizer

An optimizer determines how the weights should be updated during training. While standard Gradient Descent updates weights using a fixed rule, advanced optimizers like **Adam**, **RMSProp**, and **AdamW** adapt the updates for faster and more stable convergence.

**Goal:** Efficiently update model parameters during training.


# Understanding Epoch, Batch and Iteration

Suppose you have:

* **Dataset Size:** 10,000 samples
* **Batch Size:** 100 samples

Then:

* Number of iterations per epoch = 10,000 ÷ 100 = **100**
* If you train for **5 epochs**, the model performs **500 iterations** in total.

**Summary:**

| Term       | Meaning                                               |
| ---------- | ----------------------------------------------------- |
| Sample     | One training example                                  |
| Batch      | A subset of training samples processed together       |
| Batch Size | Number of samples in one batch                        |
| Iteration  | One parameter update after processing a batch         |
| Epoch      | One complete pass through the entire training dataset |


# Dataset Splitting

A typical dataset is divided into three parts:

```text
Entire Dataset
      │
      ├────────► Training Set (70–80%)
      │            Learn model parameters
      │
      ├────────► Validation Set (10–15%)
      │            Tune hyperparameters and monitor training
      │
      └────────► Test Set (10–15%)
                   Evaluate final model performance
```

### Training Set

Used to learn the model's parameters (weights and biases).

### Validation Set

Used during training to tune hyperparameters, detect overfitting, and select the best model.

### Test Set

Used only after training is complete to assess how well the model generalizes to unseen data.


# Why These Concepts Matter for Fine-Tuning

Fine-tuning follows the same learning process as training a neural network from scratch:

* The model performs **forward propagation** to generate predictions.
* A **loss function** measures the error.
* **Backpropagation** computes gradients.
* An **optimizer** updates the model's weights.
* This process repeats across multiple **batches** and **epochs** until the model adapts to the new task.

The key difference is that fine-tuning starts with a **pre-trained model** rather than randomly initialized weights, allowing it to learn new tasks efficiently with much less data and computation.


# Key Takeaways

* Neural networks learn by repeatedly making predictions, measuring errors, and updating their parameters.
* **Forward propagation** generates predictions, while **backpropagation** computes how to improve them.
* **Loss functions** quantify prediction errors, and **optimizers** minimize those errors.
* **Learning rate**, **batch size**, and **epochs** are critical hyperparameters that influence training efficiency and model performance.
* Proper dataset splitting into **training**, **validation**, and **test** sets ensures reliable model development and evaluation.
* These concepts form the mathematical and algorithmic foundation for understanding how LLM fine-tuning techniques such as **LoRA**, **QLoRA**, and **Supervised Fine-Tuning (SFT)** work.
